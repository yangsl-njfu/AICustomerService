# AI 客服系统功能测试文档

## 一、Agent 功能测试（推荐 Agent）

所有推荐相关需求统一由 Agent 处理，Agent 根据用户消息自动选择策略：
- 简单搜索：调 1 次工具，快速返回
- 个性化推荐：调 1 次浏览历史工具
- 复杂选题决策：多轮工具调用 + 推理

### 场景 1：简单关键词搜索（Agent 调 1 次 search_projects）

**测试输入：**
```
推荐几个python相关的毕设项目
``

---

### 场景 2：个性化推荐（Agent 调 1 次 get_personalized_recommendations）

**测试输入：**
```
有什么推荐的吗
```

**预期行为：**
- 意图识别为"推荐"
- Agent 调用 get_personalized_recommendations 查看浏览历史
- 基于历史推荐，或推荐热门商品

**验证点：**
- [ ] 日志显示 `🛠️ Agent 调用工具: get_personalized_recommendations`
- [ ] 有浏览历史时返回个性化推荐
- [ ] 无浏览历史时返回热门商品

---

### 场景 3：复杂选题决策（Agent 多轮推理）

**测试输入：**
```
我是计算机专业大四的，想做一个和医疗相关的毕设，最好用Java，预算500以内
```

**预期行为：**
- 意图识别为"推荐"
- Agent 进行多轮推理：
  1. 调用 search_projects 搜索医疗+Java 项目
  2. 调用 get_project_detail 查看候选项目详情
  3. 可能调用 check_tech_stack_match 检查技术匹配
  4. 综合分析后给出推荐理由

**验证点：**
- [ ] 日志显示多轮 `🔄 Agent 第 N 轮推理`（N >= 2）
- [ ] 工具调用链包含 search_projects → get_project_detail
- [ ] 最终推荐包含理由（技术匹配、难度适中、价格合适等）
- [ ] 推荐项目在 500 元预算内

---

### 场景 4：预算不足

**测试输入：**
```
我想做一个Java的电商系统，预算100块
```

**预期行为：**
- Agent 搜索后发现预算内项目很少或没有
- 提示预算不足，建议调整

**验证点：**
- [ ] Agent 调用了 search_projects
- [ ] 回复中提到预算限制
- [ ] 提供调整建议（提高预算或换方向）

---

## 二、Workflow 功能测试

### 1. 问答 (QA Flow)

**测试输入：**
```
你们平台是做什么的？
```

**预期行为：**
- 识别为"问答"意图
- 跳过工具调用，直接回答

**验证点：**
- [ ] 日志显示 `⚡ 跳过工具调用 (意图: 问答)`
- [ ] 快速响应，回答平台介绍

---

### 2. 商品咨询 (Product Inquiry)

**测试输入：**
```
这个医院管理系统用什么技术栈？
```

**预期行为：**
- 识别为"商品咨询"意图

- [ ] 准确回答技术栈

### 4. 订单查询 (Order Query)

**预期行为：**
- 识别为"订单查询"意图

## 五、测试注意事项

1. **Agent vs Workflow 区分**：所有"推荐"意图走 Agent（跳过 function_calling），其他意图走 Workflow
2. **Agent 工具调用次数**：简单搜索 1 次，复杂决策 2-4 次，看日志中 `🔄 Agent 第 N 轮推理` 的次数
3. **流式输出**：Agent 的工具调用阶段会显示状态提示（如"正在搜索项目..."），最终回答逐字输出
4. **上下文记忆**：测试同一 session 内的上下文保持
